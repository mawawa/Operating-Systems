OS Security and Protection II

Lecture Notes

There are many ways to secure and protect the operating system. These solutions include user training , using technology and using better software. It is important to understand that the more leyers of security the better.
The first ways to secure the operating systems is to have a security policy.

Security policy

a security policy example may state the following :

  • All outside application must go through a code review before being depoyed
  • All users should not share their password

Security policy is able to direct the users on the permissable actions, what is required and what is not to be done. The policy must be reviewed and updated periodically

Intrusion Prevention

Intrusion prevention is concerened with detecting intrusions that would be both attemted intrusion and successfull intrusion. There are many ways to conduct intrusion prevention

Signature based detection
This attempts to characterize dangerous attempts
Anomaly based detection
This attempts to detect all the strange behaiviour.

The main differnce between the two is that Anomally detection can detect previously unknown. intrusions while signature based prevention detects previously know intrusions.

Auditing, Account and Logging

Authentication and authorization failures can tell us a lot about intrusion attempts.

Firewall to protect system networks

A firewall is intalled in order to connect a computer into an untrustwothy network. The firewall seperates the computer from the network. The firewall can also limit communication based on source and destination and ports and direction. There are different types of firewals :

  • A personal firewall
  • An application proxy firewall.
  • An XML firewall -this firewall is used for analyzing XML trafic and also blocking disallowed or malformed XML
  • System Calls firewall - This firewal sits between the computer programs and the kernel that maked the system calls.

Protection

There are several reasons why protection is needed in a system. The main reason is to prevent users michevious behavior. The time-tested guiding principle is the principle of least privilage. that is a program is only given just enough privillages to perform their task.
The principle of complatmentalization also is a means of security .
