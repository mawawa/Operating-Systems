Introduction

Lecture Notes

A process is a program in execution. The status of a process is held by the program counter and the contents of the memory registers. A process is held in memory in the multiple section.

Text Section
This section holds the code to be executed.
Data Section
This section holds the programs global variables.
Heap Section
This section is the memory that is dynamically allocated to the program during execution.
Stack Section
This section is temporal storage for invoking functions, that is the parameters and return values and local variables.

The size of the data and the text section are fixed as their sizes do not change in the execution of the program. On the other hand the heap and the stack section change as the program runs. Although the stack and the heap sections grow during execution the operating system should make sure that the two do not overlap.

Program vs Process

A program is just information with the list of instruction stored in a location while a process is and active entity that has a program counter specifying the next instruction to execute along with allocates system resources. A program becomes a process when its executable file is loaded into memory.

Process States

A process can be in one of 5 states :
New
The process is being created.
Running
The instructions in the text part of the process are being executed.
Waiting
The process is wai
Ready
The process is waiting to be assigned to a processor.
Terminated
The process has finished execution

Process Control Block

A process in the operating system is represented with ta process control block. The PCB is made up of the following section:

Process State
The process state is show which of the five states the process is in, that is new , running , waiting , terminated, ready
Program Counter
Ths program counter host the location of the next instruction to be executed.
CPU Registers
The registers vary in number and type, depending on the computer architecture
CPU Scheduling
This part contains the information about process priority, pointer to shceduling queues, and other scheduling parameters.
Memory management Information
This information may include such items as the value of the base and limit registers and the page tables, or the segment tables, depending on the memory system used by the operating system
Accounting Information
This information includes the amount of CPU and real time used, time limits, account numbers, job or process numbers, and so on.
I/O Status Information
This information includes the list of I/O devices allocated to the process, a list of open files, and so on.

1. Process Creation

the state of the process here is new. During the creation of a process a process my also create many other processes. The process that creates the other processes is called a parent process. The processes are uniquely identified with their PID the id process identifier.
For Example the process with PID 1 in a linux system will be the parent of all the processes which that particular user will create. The system process always has a PID of 1.

2. Process Termination

A process terminates when it is finished executing its last instruction and asked the operating systems to delete it by executing calling exit() system call. The process may return a status value to the parent process via the wait() system call. All the resources given to the process are returned to the operating system. A process may terminate another process by invoking the TerminateProcess() system call. this system call can only be called by a parent process to its child processes. Cascading termination states that some systems donot allow childe processes to exist without the parent processes existing. Thus if the parent processs terminates then all its children processes are terminated.

Process Classification

Below is the hierarchy of the processes
Foreground process
This is the process that is currenly visible on the screen
Visible process
A process that is not directly visible on the foreground but that is performing an activity that the foreground process is referring to
Service process
this is a process similar to a background process but differs in that its execution is apparent to the user , like streaming music.
Background process
this is a process that executes in the background and is not apparent to the user.
Empty process
this is a process that holds no active components.

Threads

threads allow processes to have multiple execution streams. That is in a word processor one may have a Thread for typing and then another thread for spellchecking. This feature is most beneficial to systems with multi cores where each thread can executed on a different core allowing for parallel execution.

Inter process communication

Processes of a operating system may be either independent of cooperating processes. There are three main reasons for having cooperating process :

Sharing information
processes may be interested in the same information and thus sharing of infomration for example in instances of copy and paste.
Computation speedup
If we want a particular task to run faster, we must break it into subtasks, each of which will be executing in parallel with the others.
Modularity
We may want to construct the system in a modular fashion, dividing the system functions into separate processes or threads.

There are two fundamental IPC these are

  1. Shared Memory
  2. Message passing

Process Scheduling

Multiprogramming - This is most concerned with making sure that the cpu always has some process that is executing.
Time Sharing- this is most concerned with how to switch a CPU core among processes so frequently that users can interact with each program while it is running.
To meet these objectives the use of a process scheduler is needed. The process scheduler is able to select an available process for program execution in the core. Each CPU core can run one proces at a time. The number of processes currently in memory is known as a degree of multiprogramming.
Processes in multiprogramming and timesharing can be described in one of two ways.:
I/O bound Processor- This is the process that spends most of its time doing input output than it spends calculating.
CPU-Bound Processes - This is a process that spends most of its time doing computation.

Processes's PCB are loaded into a queue that is in form of a linked list.

Another way to visualize process scheduling is through the use of queue diagrams.

The CPU scheduler runs approximately every 100 milliseconds although usually it is much faster. If a process runs logner then it is forcibly stopped usually in the situation of I/O Bound processes which would be held by waiting for and I/O request.

Swapping

Context Switch

Context switch helps in the situation the a system call is made of a system interupt which causes the shceduler to save the state that process the PCB loads the next one to the PCB and the executes that one.
