OS Security and Protection

Lecture Notes

Security involves guarding computer resources against unauthorized access, malicious destruction or alteration, and accidental introduction of inconsistency
Computer resources include the information stored in the system (both data and code), as well as the CPU, memory, secondary storage, tertiary storage, and networking that compose the computer facility
Security violation can be catergorized as

  1. Intentional
  2. Accidental

Forms of malicious security violations

1. Breach of integrity
This type of violation involves unauthorized reading of data
2. Breach of integrity
This violation involves unauthorized modification of data
Theft of Service
This violation involves unauthorized use of resources.
Denial of Service
This violation involves preventing legitimate use of the systems.

The most common security problem is masquerading, in which one participant in a communication pretends to be someone else. Another common attack is to replay a captured exchange of data.

Four Measures to protect a system

1.Physical
The site of sites containing the computer systesm must be physically secured against entry by intruders
2. Network
Most contempooryary computer systems form servers to mobile devices to internet of things are networked.
3. Operating System
The operation systems and its built-in set of application and services comprice a huge code base that may hobor vulnerabilities.
4. Application
Third party application may also pose risks especially if they posses significant privilatges.

Program Threats

1. Malware
Malware is a software sesigned to ecpoit, disable or damage computer systems. An example of malware is on the Trojan horce clissified as spyware. Another form of malware is the Ransomware
2. Code Injection
Most software is not malicious, but it can nonetheless pose serious threats to security due to a code-injection attack, in which executable code is added or modified.
Viruses and Worms
A virus is a fragment of code embedded in a legitimate program. They usually come from spam email and phising attacks.

System Network Threats

Program threats, by themselves, pose serious security risks. • But those risks are compounded by orders of magnitude when a system is connected to a network. • Worldwide connectivity makes the system vulnerable to worldwide attacks.
For that reason, hackers frequently launch attacks from zombie systems— • Zombie systems are independent systems or devices that have been compromised by hackers but that continue to serve their owners while being used without the owners’ knowledge for nefarious purposes including denial-of-service attacks and spam relay.

Common network attacks includes :

  1. Sniffing
  2. Dos
  3. port scanning

Cryptography as a Security tool

It is generally considered infeasible to build a network of any scale in which the source and destination addresses of packets can be trusted in this sense. • Therefore, the only alternative is somehow to eliminate the need to trust the network. • This is the job of cryptography.

There are two main types of encryption algorithms:

  1. Asymetric Encrpytion
  2. Symetric Encryption

Authentication

• An authentication algorithm using symmetric keys consists of the following components:
• A set K of keys.
• A set M of messages.
• A set A of authenticators.
• A function S : K → (M → A).
• A function V : K → (M × A → {true, false}). That is, for each k ∈ K, Vk is a function for verifying authenticators on messages.
There are 2 main varieties of authentication algorithms.

1. Message-authentication code
Uses symetric encyption. here a cryptographic checksum is generated from the message using a secret key
2. Digital-signature algorythm
The authenticators are called digital signatures. Digital signatures are very usefull in that they enable anyone to veryfy the authenticity of the message.

Users Authentication is based on one or more of three things :

  1. The users possesion of something
  2. The users knowledge of something
  3. An attribute of the user eg a finger print.

Passwords

The most common approach to authenticating a user is to use a password. which falls under something the user knows. There are three ways to break a password

  1. to know the user and use their common password since all too frequently users use names of their cats, siblings spouses.
  2. Another way is to use brute force.
  3. A third way is to use brute force.

Advancements have been made to include and OTP which is a one time password. These are most commonly used in two factor authentication.
